# Шпаргалка: Модель OSI (Open Systems Interconnection)

## Что такое OSI модель?

**OSI модель** — это концептуальная модель, описывающая, как данные передаются по сети через **7 уровней**.

**Зачем нужна:**
- Стандартизация сетевых протоколов
- Диагностика сетевых проблем
- Понимание, как работают сети

---

## 7 уровней OSI (сверху вниз)

| № | Уровень | Английское название | Что делает | Примеры |
|---|---------|---------------------|------------|---------|
| 7 | Прикладной | Application | Взаимодействие с приложениями | HTTP, DNS, FTP, SSH |
| 6 | Представления | Presentation | Шифрование, сжатие, кодирование | SSL/TLS, JPEG, ASCII |
| 5 | Сеансовый | Session | Управление сеансами связи | NetBIOS, RPC |
| 4 | Транспортный | Transport | Доставка данных (надёжность) | TCP, UDP |
| 3 | Сетевой | Network | Маршрутизация (IP адреса) | IP, ICMP, routing |
| 2 | Канальный | Data Link | Передача между устройствами (MAC) | Ethernet, Wi-Fi, switches |
| 1 | Физический | Physical | Передача битов (электричество, свет) | Кабели, Wi-Fi сигнал |

## Подробное описание уровней

### Уровень 7: Прикладной (Application Layer)

**Что делает:**
- Взаимодействие с пользовательскими приложениями
- Предоставляет сетевые сервисы приложениям

**Протоколы:**
- **HTTP/HTTPS** — веб-страницы
- **DNS** — преобразование доменов в IP
- **FTP** — передача файлов
- **SMTP** — отправка email
- **SSH** — удалённый доступ

**Инструменты:**
```bash
curl https://google.com        # HTTP запрос
dig google.com                 # DNS lookup
ssh user@server                # SSH подключение
```

**Пример проблемы:**
- "Сайт не открывается" → проверить HTTP код (`curl -I`)

---

### Уровень 6: Представления (Presentation Layer)

**Что делает:**
- Шифрование/дешифрование данных
- Сжатие данных
- Преобразование форматов (кодировки)

**Протоколы:**
- **SSL/TLS** — шифрование HTTPS
- **JPEG, PNG** — форматы изображений
- **ASCII, UTF-8** — кодировки текста

**Пример:**
- Браузер получает зашифрованные данные (HTTPS) → SSL расшифровывает → показывает текст

**Инструменты:**
```bash
openssl s_client -connect google.com:443    # Проверить SSL сертификат
```

**Пример проблемы:**
- "Сертификат SSL истёк" → проблема на уровне 6

---

### Уровень 5: Сеансовый (Session Layer)

**Что делает:**
- Устанавливает, управляет и завершает сеансы связи
- Синхронизация данных
- Контрольные точки (checkpoints)

**Протоколы:**
- **NetBIOS** — сетевые сеансы Windows
- **RPC** — удалённый вызов процедур
- **PPTP** — VPN туннели

**Пример:**
- Ты открыл SSH сессию → работаешь → закрыл → сеанс завершён

**Пример проблемы:**
- "SSH сессия обрывается через 5 минут" → проблема на уровне 5

---

### Уровень 4: Транспортный (Transport Layer)

**Что делает:**
- Доставка данных от приложения к приложению
- Разбивает данные на сегменты
- Контроль ошибок и надёжности

**Протоколы:**
- **TCP** — надёжная доставка (с подтверждением)
- **UDP** — быстрая доставка (без подтверждения)

**Порты:**
- HTTP: 80
- HTTPS: 443
- SSH: 22
- MySQL: 3306

**Инструменты:**
```bash
ss -tuln                       # Показать открытые порты
nc -zv google.com 443          # Проверить доступность порта
telnet google.com 80           # Подключиться к порту
```

**Пример проблемы:**
- "Порт 80 закрыт" → проверить firewall (`ufw status`)

**Разница TCP vs UDP:**
| TCP | UDP |
|-----|-----|
| Надёжный (с подтверждением) | Быстрый (без подтверждения) |
| HTTP, SSH, FTP | DNS, видео-стриминг, игры |
| Медленнее | Быстрее |

---

### Уровень 3: Сетевой (Network Layer)

**Что делает:**
- Маршрутизация пакетов между сетями
- Определяет путь от источника к получателю
- Использует IP адреса

**Протоколы:**
- **IP (IPv4, IPv6)** — адресация
- **ICMP** — ping, traceroute
- **Routing protocols** — BGP, OSPF

**Инструменты:**
```bash
ping google.com                # Проверить доступность хоста
traceroute google.com          # Показать маршрут пакетов
mtr google.com                 # Комбинация ping + traceroute
ip route                       # Показать таблицу маршрутизации
```

**Пример проблемы:**
- "Не могу достучаться до сервера" → `ping` не проходит → проблема маршрутизации

**IP адреса:**
- **IPv4:** 192.168.1.1 (32 бита)
- **IPv6:** 2001:0db8:85a3::8a2e:0370:7334 (128 бит)

---

### Уровень 2: Канальный (Data Link Layer)

**Что делает:**
- Передача данных между соседними устройствами в одной сети
- Использует MAC адреса
- Обнаружение и исправление ошибок

**Протоколы:**
- **Ethernet** — проводные сети
- **Wi-Fi (802.11)** — беспроводные сети
- **ARP** — преобразование IP в MAC

**Устройства:**
- **Switch** — коммутатор (работает на уровне 2)
- **Bridge** — мост

**Инструменты:**
```bash
ip link show                   # Показать сетевые интерфейсы
arp -a                         # Показать ARP таблицу (IP → MAC)
ethtool eth0                   # Информация о сетевом интерфейсе
```

**MAC адрес:**
- Пример: `00:1A:2B:3C:4D:5E`
- Уникальный адрес сетевой карты

**Пример проблемы:**
- "Кабель подключён, но сеть не работает" → проверить `ip link show` (интерфейс UP?)

---

### Уровень 1: Физический (Physical Layer)

**Что делает:**
- Передача битов (0 и 1) через физическую среду
- Электрические сигналы, световые импульсы, радиоволны

**Среда передачи:**
- **Кабели:** Ethernet (витая пара), оптоволокно
- **Беспроводная:** Wi-Fi, Bluetooth, 4G/5G

**Устройства:**
- **Hub** — концентратор (повторяет сигнал)
- **Repeater** — усилитель сигнала
- **Кабели, разъёмы**

**Инструменты:**
```bash
ethtool eth0                   # Скорость, дуплекс
dmesg | grep eth               # Логи сетевой карты
```

**Пример проблемы:**
- "Сеть не работает" → кабель не подключён, сетевая карта сломана

---

## Как данные проходят через OSI модель

### Отправка данных (7 → 1)

**Пример:** Ты открываешь `https://google.com`

1. **Уровень 7 (Прикладной):** Браузер формирует HTTP запрос `GET / HTTP/1.1`
2. **Уровень 6 (Представления):** SSL шифрует данные
3. **Уровень 5 (Сеансовый):** Устанавливается сеанс связи
4. **Уровень 4 (Транспортный):** TCP разбивает данные на сегменты, добавляет порт (443)
5. **Уровень 3 (Сетевой):** IP добавляет адреса (твой IP → IP Google)
6. **Уровень 2 (Канальный):** Ethernet добавляет MAC адреса
7. **Уровень 1 (Физический):** Данные передаются по кабелю/Wi-Fi как электрические сигналы

### Получение данных (1 → 7)

Сервер Google получает данные и проходит уровни **снизу вверх** (1 → 7).

## Диагностика проблем по уровням OSI

**Правило:** Проверяй **снизу вверх** (1 → 7).

### Проблема: "Сайт не открывается"

**Шаг 1: Уровень 1 (Физический)**
```bash
ip link show
# Проверить: интерфейс UP? Кабель подключён?
```

**Шаг 2: Уровень 2 (Канальный)**
```bash
arp -a
# Проверить: есть ли MAC адрес шлюза?
```

**Шаг 3: Уровень 3 (Сетевой)**
```bash
ping 8.8.8.8
# Проверить: доступен ли интернет?
```

**Шаг 4: Уровень 4 (Транспортный)**
```bash
nc -zv google.com 443
# Проверить: открыт ли порт 443?
```

**Шаг 5: Уровень 7 (Прикладной)**
```bash
curl -I https://google.com
# Проверить: HTTP код 200?
```

## TCP/IP модель vs OSI модель

**TCP/IP модель** — упрощённая версия OSI (4 уровня вместо 7).

| OSI (7 уровней) | TCP/IP (4 уровня) |
|-----------------|-------------------|
| 7. Прикладной | 4. Прикладной |
| 6. Представления | ↑ |
| 5. Сеансовый | ↑ |
| 4. Транспортный | 3. Транспортный |
| 3. Сетевой | 2. Интернет |
| 2. Канальный | 1. Канальный |
| 1. Физический | ↑ |

**На практике используется TCP/IP**, но **на собеседованиях спрашивают OSI**.

## Типичные вопросы на собеседованиях

### Вопрос 1: "Объясни OSI модель"

**Ответ:**
OSI модель — это 7 уровней, описывающих передачу данных по сети:
1. Физический — передача битов (кабели, Wi-Fi)
2. Канальный — передача между устройствами (MAC адреса, Ethernet)
3. Сетевой — маршрутизация (IP адреса, ping)
4. Транспортный — доставка данных (TCP/UDP, порты)
5. Сеансовый — управление сеансами
6. Представления — шифрование, сжатие (SSL/TLS)
7. Прикладной — взаимодействие с приложениями (HTTP, DNS, SSH)

---

### Вопрос 2: "На каком уровне работает HTTP?"

**Ответ:** Уровень 7 (Прикладной)

---

### Вопрос 3: "На каком уровне работает TCP?"

**Ответ:** Уровень 4 (Транспортный)

---

### Вопрос 4: "На каком уровне работает IP?"

**Ответ:** Уровень 3 (Сетевой)

---

### Вопрос 5: "На каком уровне работает switch?"

**Ответ:** Уровень 2 (Канальный) — использует MAC адреса

---

### Вопрос 6: "На каком уровне работает router?"

**Ответ:** Уровень 3 (Сетевой) — использует IP адреса

---

### Вопрос 7: "Пользователь не может открыть сайт. Как диагностировать?"

**Ответ:**
Проверяю снизу вверх:
1. **Уровень 1:** Кабель подключён? (`ip link show`)
2. **Уровень 3:** Интернет работает? (`ping 8.8.8.8`)
3. **Уровень 3:** DNS работает? (`dig google.com`)
4. **Уровень 4:** Порт открыт? (`nc -zv google.com 443`)
5. **Уровень 7:** HTTP код? (`curl -I https://google.com`)

---

### Вопрос 8: "В чём разница между TCP и UDP?"

**Ответ:**
- **TCP:** Надёжный (с подтверждением), медленнее. Используется для HTTP, SSH, FTP.
- **UDP:** Быстрый (без подтверждения), ненадёжный. Используется для DNS, видео-стриминг, игры.

---

### Вопрос 9: "Что такое MAC адрес и на каком уровне он используется?"

**Ответ:**
MAC адрес — уникальный адрес сетевой карты (например, `00:1A:2B:3C:4D:5E`).
Используется на уровне 2 (Канальный).

---

### Вопрос 10: "Что делает команда ping и на каком уровне она работает?"

**Ответ:**
`ping` проверяет доступность хоста, отправляя ICMP пакеты.
Работает на уровне 3 (Сетевой).

## Практические примеры

### Пример 1: Открытие веб-страницы

**Действие:** Ты вводишь `https://google.com` в браузере

**Что происходит:**

1. **Уровень 7:** Браузер формирует HTTP запрос
2. **Уровень 6:** SSL шифрует данные
3. **Уровень 4:** TCP разбивает на сегменты, добавляет порт 443
4. **Уровень 3:** IP добавляет адреса (твой IP → IP Google)
5. **Уровень 2:** Ethernet добавляет MAC адреса
6. **Уровень 1:** Данные передаются по кабелю

**Сервер Google:**
1. **Уровень 1:** Получает электрические сигналы
2. **Уровень 2:** Читает MAC адреса
3. **Уровень 3:** Читает IP адреса
4. **Уровень 4:** Собирает TCP сегменты
5. **Уровень 6:** Расшифровывает SSL
6. **Уровень 7:** Обрабатывает HTTP запрос, отправляет HTML

---

### Пример 2: SSH подключение

**Действие:** `ssh user@server`

**Что происходит:**

1. **Уровень 7:** SSH клиент формирует запрос
2. **Уровень 6:** SSH шифрует данные
3. **Уровень 4:** TCP устанавливает соединение на порт 22
4. **Уровень 3:** IP маршрутизирует пакеты к серверу
5. **Уровень 2:** Ethernet передаёт данные
6. **Уровень 1:** Физическая передача

---

### Пример 3: DNS lookup

**Действие:** `dig google.com`

**Что происходит:**

1. **Уровень 7:** DNS клиент формирует запрос "Какой IP у google.com?"
2. **Уровень 4:** UDP отправляет запрос на порт 53
3. **Уровень 3:** IP маршрутизирует к DNS серверу (8.8.8.8)
4. **Уровень 2-1:** Физическая передача

**DNS сервер отвечает:** `google.com = 142.250.185.46`

```
7. Прикладной    → HTTP, DNS, SSH          → curl, dig, ssh
6. Представления → SSL/TLS, шифрование     → openssl
5. Сеансовый     → Управление сеансами     → —
4. Транспортный  → TCP, UDP, порты         → ss, nc, telnet
3. Сетевой       → IP, маршрутизация       → ping, traceroute, mtr
2. Канальный     → MAC, Ethernet, Wi-Fi    → ip link, arp
1. Физический    → Кабели, сигналы         → ethtool, dmesg
```
